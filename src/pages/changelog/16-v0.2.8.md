# v0.2.8

## Performance

- **Floating toolbar memoization**
  - Memoized the `exportProps` object in [App.tsx](cci:7://file:///Users/lucamack/Desktop/projects/quickdoc.react/src/App.tsx:0:0-0:0) and pass it into `FloatingToolbar` as a stable prop.
  - Reduces unnecessary re-renders of the memoized `FloatingToolbar` when the current page content does not change.
- **Sidebar numbering dependencies**
  - Simplified `useMemo` dependencies for sidebar numbering to match the actual config flags that influence numbering.
  - Avoids recalculations when unrelated navigation configuration changes.

## Improvements

- **Config defaults**
  - `navigation.expandAllSections` now defaults to `false` so sidebars are collapsed by default and expand contextually.
  - `navigation.enableUserSidebarWidthChange` is now `true` so users can interactively resize the sidebar.
  - `navigation.enableNumberedSidebar` is now `false` by default, disabling numbered sidebar items unless explicitly enabled.
- **Font configuration**
  - Switched the default sans font stack to system fonts only for faster initial render and fewer external font dependencies.
  - Commented out the Google Fonts `sans` entry, keeping only the `Geist Mono` example for code.
- **Border system tuning**
  - Adjusted the `radiusMap` values in `borderStyles` to better fit the overall UI scale.
  - Improved border color mapping for light and dark themes using the Tailwind color-based theme configuration.

## Internal Cleanup

- **MDX and markdown pipeline**
  - Removed or commented out noisy debug logs in the MDX compiler, parser, Vite raw MDX plugin, and markdown file loader.
  - Normalized string handling and small parsing details in `mdxParser` for more predictable behavior.
- **Theming utilities**
  - Dropped debug logging from `codeBlockStyles`, `useDarkMode`, `MarkdownContent`, and other helpers now that the theme system is stable.
